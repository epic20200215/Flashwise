# 闪学开发最高规则

本文件是闪学项目的最高开发规则。任何代码、设计、内容、提示词、运营配置和自动化代理输出都必须遵守本文件；如与其他文档冲突，以本文件为准。

## 1. 产品原则

1. 闪学是“AI 生成学习材料 + 主动回忆 + 间隔重复 + 游戏化练习”的中文学习工具，不是搜题抄答案工具，也不是面向中小学生的学科类校外培训服务。
2. 产品目标是帮助用户把已有学习材料转化为可复习、可测验、可长期记忆的结构化知识。
3. 所有 AI 输出必须鼓励理解过程、分步提示和自我作答，不得默认直接给作业最终答案。
4. 可以借鉴 Gizmo 的产品机制和扁平化简约风格，但不得复制 Gizmo 的商标、角色、素材、布局细节、专属文案、图标或其他受保护表达。

## 2. 目标用户与边界

1. MVP 优先服务大学生、职业考试、自学成人和高中以上用户。
2. 面向未成年人时必须启用年龄分层、监护人同意、使用时长控制、内容安全过滤和防沉迷策略。
3. 义务教育阶段内容只做“用户自有资料的复习工具”和“公益性学习辅助”，避免承诺提分、排名焦虑营销、真人培训课售卖和学科类培训包装。

## 3. 合规红线

1. 在中国大陆上线前必须完成 App 备案、隐私合规、必要的算法/生成式 AI 服务评估与备案判断、内容安全机制、投诉举报入口。
2. 用户上传的教材、讲义、图片、音频、视频和链接只可用于用户授权的学习材料生成，不得默认用于训练模型。
3. 不收集非必要个人信息；儿童个人信息必须取得监护人同意并提供删除、更正、撤回同意路径。
4. 所有公开分享、公共卡组、排行榜昵称、头像、评论、AI 生成内容都必须进入内容安全审核链路。
5. 生成内容必须标注“AI 生成，仅供学习参考”，并提供纠错、举报和重新生成入口。

## 4. 学习体验规则

1. 每个学习闭环必须尽量短：导入资料、生成卡片、开始测验、得到解释、安排复习。
2. 题目类型至少支持：选择题、填空题、正误题、配对题、排序题、翻卡自评、手写/输入题。
3. 错题反馈必须解释原因、引用来源片段、给出下一步练习，而不是只显示对错。
4. 间隔重复必须以卡片掌握度、错误类型、考试日期和用户学习强度为输入，不得只做固定日期提醒。
5. 游戏化只能服务学习：连胜、经验、生命值、排行榜、组队挑战都不得诱导未成年人沉迷或过度消费。

## 5. 设计规则

1. 美术风格采用扁平化、简约、明亮、低噪音的学习工具风格；优先清晰、速度感和可读性。
2. 默认界面不得营销化，不做大面积插画首页；打开 App 后应快速进入“继续复习 / 导入资料 / 今日任务”。
3. 色彩使用白底、深灰文字、绿色学习高亮、少量蓝色行动按钮和黄色/橙色反馈点缀，避免单一紫蓝渐变或厚重拟物。
4. 组件必须适配中文长文本，不允许按钮、卡片、题干、解释文本溢出。

## 6. 技术规则

1. 代码优先模块化：导入解析、卡片生成、题目生成、记忆调度、AI 导师、审核、支付、账号、社区各自独立。
2. 任何 AI 生成学习内容都必须保存来源映射，支持追溯到用户上传资料的段落、页码或时间戳。
3. 对外部模型调用必须走统一网关，记录模型版本、提示词版本、安全过滤结果、成本和延迟。
4. 支持降级：AI 失败时用户仍可手动建卡、复习已有卡片、查看本地缓存。
5. 所有关键接口必须包含权限校验、速率限制、审计日志和异常告警。

## 7. 质量门槛

1. MVP 不以功能堆叠为目标，必须先把“导入到可学卡片”与“每天复习有效”做稳定。
2. 每个版本必须跑通核心路径：注册、导入、生成、编辑、测验、解释、复习计划、分享、删除数据。
3. 涉及未成年人、支付、公开社区、AI 输出和用户数据的改动必须有额外测试与人工审核。
4. 不得为了短期增长牺牲隐私、合规、内容安全或学习质量。
