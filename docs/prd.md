# 闪学 MVP PRD

## 1. MVP 目标

在 12 周内做出可内测的移动端 App：用户可以注册、导入中文资料、自动生成卡片、编辑卡片、完成多题型测验、获得解释、按间隔重复复习，并能创建小组分享卡组。

## 2. MVP 范围

### 必做

- 手机号登录、微信登录预留。
- 文件/图片/文本导入。
- OCR、文档解析、AI 生成卡片。
- 卡片审阅、编辑、删除、合并。
- 选择题、填空题、翻卡自评、正误题。
- 答题解释、来源片段引用。
- 间隔重复调度。
- 首页今日复习。
- XP、连胜、基础好友/小组分享。
- 隐私设置、数据删除、投诉举报。
- 内容安全审核与 AI 输出过滤。

### 暂不做

- 全量公共卡组市场。
- 直播课、真人辅导、付费课程。
- 大规模学校后台。
- 复杂陌生人社交。
- iPad/桌面端深度适配。

## 3. 用户故事

1. 作为大学生，我上传老师的 PPT，希望 1 分钟内生成可复习卡片。
2. 作为考研用户，我希望系统按考试日期安排每天复习任务。
3. 作为医学生，我希望答错后看到原文出处和解释，避免 AI 胡编。
4. 作为学习小组组长，我希望分享卡组并让成员一起挑战。
5. 作为未成年用户监护人，我希望控制使用时长、查看隐私说明并删除数据。

## 4. 核心流程

### 注册激活

1. 用户手机号注册。
2. 选择身份：大学生、职业考试、自学、高中、其他。
3. 选择目标考试或自定义目标。
4. 进入首页，展示“导入第一份资料”。

### 导入生成

1. 用户选择文件/图片/文本。
2. 系统解析，显示资料标题、页数、字数。
3. 用户选择生成模式：概念记忆、考试测验、词汇、问答理解。
4. AI 生成卡片与来源映射。
5. 用户审阅并保存。

### 学习测验

1. 用户进入卡组，点击开始。
2. 系统按掌握度选题。
3. 用户答题。
4. 系统显示对错、解释、来源、下一题。
5. 会话结束后显示掌握率、XP、下次复习安排。

## 5. 验收标准

- 10 页以内 PDF 在 90 秒内完成解析和首批卡片生成。
- 生成卡片至少 80% 能定位到来源片段。
- 用户可删除自己的资料、卡片、账号数据。
- AI 输出经过安全过滤；公开分享内容必须审核。
- 核心学习流程离线缓存已有卡片，网络恢复后同步。
